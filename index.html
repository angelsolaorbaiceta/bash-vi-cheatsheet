Setup & Configuration
~/.inputrc — recommended config
# Enable vi mode globally (for all readline apps: bash, psql, gdb…)
set editing-mode vi

# Show current mode in the prompt: (cmd) / (ins)
set show-mode-in-prompt on
set vi-ins-mode-string "\1\e[6 q\2"   # beam cursor in insert
set vi-cmd-mode-string "\1\e[2 q\2"   # block cursor in normal

# Keep Ctrl-R in insert mode (incremental search)
$if mode=vi
  set keymap vi-insert
  Control-r: reverse-search-history
  Control-s: forward-search-history
  Control-l: clear-screen
$endif
Note: The v command in normal mode opens the current command in $EDITOR (falls back to vi). For complex multi-line commands this is indispensable — edit freely, then :wq to execute. Also works in zsh vi mode with edit-command-line.
Tip: Ctrl-r in insert mode is the fastest way to search history — it's muscle memory worth keeping. The /pattern search in normal mode is regex-based and great for finding long past commands. Combine with n/N to navigate matches. Also consider bind -x '"\C-x\C-e": edit_and_execute_command' in .bashrc as an alternative to v.
